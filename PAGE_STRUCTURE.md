# 页面结构设计

## 区域排列顺序

1. **Hero** - 首屏大图+CTA
2. **ToolSection** - 核心工具区
3. **Examples** - 示例展示
4. **HowItWorks** - 使用步骤
5. **WhatIs** - 产品介绍
6. **Differentiation** - 差异化优势
7. **UseCases** - 使用场景
8. **FAQ** - 常见问题
9. **AITools** - 更多工具推荐

## 排列理由

| 位置 | 区域 | 理由 |
|-----|------|------|
| 1 | Hero | 首屏标配，建立第一印象 |
| 2 | ToolSection | 工具类产品核心，用户最想要的东西放最前 |
| 3 | Examples | 紧跟工具，展示效果，"用了会变成这样" |
| 4 | HowItWorks | 用完/看完示例后，教不熟悉的用户怎么用 |
| 5 | WhatIs | SEO 需要，但优先级低于使用体验 |
| 6 | Differentiation | 建立信任，说明优势 |
| 7 | UseCases | 场景拓展，次要信息 |
| 8 | FAQ | 解答疑问，转化前最后一道关卡 |
| 9 | AITools | 页面底部导流，不影响主产品体验 |

## 设计逻辑

- **上半部分 - 行动区**：用工具 → 看效果 → 学方法
- **下半部分 - 说服区**：是什么 → 为什么好 → 能干嘛 → 还有问题吗 → 其他推荐
